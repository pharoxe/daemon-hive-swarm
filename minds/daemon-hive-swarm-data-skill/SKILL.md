# Daemon Hive Swarm Data Skill

Use this Skill to let an Animoca Mind query Daemon Hive's anonymized Hive P2P dataset manifest and request privacy-preserving summaries from the swarm.

## What It Does

- Reads enabled dataset manifests advertised by Daemon Hive peers.
- Requests anonymized aggregate summaries for Android sensor, device performance, network quality, and medical-report datasets.
- Returns only schema-level, aggregate, or hash-linked results.
- Refuses raw sensor streams, exact locations, device identifiers, report text, names, dates of birth, medical IDs, facilities, or wallet private data.
- Prepares Mantle benchmark log payloads so each Mind-access decision and outcome can be recorded by hash.

## Required Connection

Create a Minds Connection named `daemon-hive-swarm` with:

- `baseUrl`: HTTPS endpoint or local tunnel for the Daemon Hive gateway.
- `apiKey`: gateway key generated by the Daemon Hive operator.
- `mantleBenchmarkContract`: deployed `MantleTuringBenchmark` contract address.
- `erc8004AgentId`: the Daemon Hive ERC-8004 identity NFT id.

The Builder API key is only used by Minds CLI/client tooling via `MINDS_BUILDER_API_KEY`; do not place it in this Skill.

## Allowed Actions

- `list_datasets`: show available anonymized datasets and privacy rules.
- `query_dataset_summary`: request aggregate summaries for selected datasets.
- `prepare_mantle_benchmark`: generate the hash payload for a Mantle benchmark event.

## Playbook

1. Inspect available datasets and their privacy rules.
2. Ask the user which dataset category and time window they want.
3. Query only anonymized aggregates.
4. Summarize gaps and confidence.
5. Prepare a Mantle benchmark hash payload for the Mind action and outcome.
6. Refuse requests for raw, identifying, or medical free-text data.
